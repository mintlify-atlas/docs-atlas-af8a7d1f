## Overview

Environment variables are used to configure the application for different environments (development, staging, production) without changing code. They are typically stored in `.env` files.

<Note>
  Environment variables prefixed with `NEXT_PUBLIC_` are exposed to the browser. All other variables are only available on the server side.
</Note>

---

## API Configuration

### NEXT_PUBLIC_API_URL

Base URL for the backend API.

<ParamField path="NEXT_PUBLIC_API_URL" type="string" required={true}>
  The base URL for all API requests
</ParamField>

#### Example Values

<CodeGroup>
```bash Development
NEXT_PUBLIC_API_URL=http://localhost:3001/api
```

```bash Staging
NEXT_PUBLIC_API_URL=https://api-staging.example.com
```

```bash Production
NEXT_PUBLIC_API_URL=https://api.example.com
```
</CodeGroup>

## Usage in Code

### Accessing Environment Variables

<CodeGroup>
```typescript Client-Side
// Only NEXT_PUBLIC_ prefixed variables are available
const apiUrl = process.env.NEXT_PUBLIC_API_URL;
const gaId = process.env.NEXT_PUBLIC_GA_ID;

console.log('API URL:', apiUrl);
```

```typescript Server-Side
// All environment variables are available
const apiUrl = process.env.NEXT_PUBLIC_API_URL;
const dbUrl = process.env.DATABASE_URL;
const jwtSecret = process.env.JWT_SECRET;

console.log('Database:', dbUrl);
```

```typescript Type-Safe Access
// Define types for environment variables
interface Env {
  NEXT_PUBLIC_API_URL: string;
  NEXT_PUBLIC_GA_ID?: string;
  DATABASE_URL: string;
  JWT_SECRET: string;
}

const env = process.env as unknown as Env;

const apiUrl = env.NEXT_PUBLIC_API_URL;
```
</CodeGroup>

## Environment Files

### File Structure

<CodeGroup>
```bash .env.local
# Local development overrides (not committed to git)
NEXT_PUBLIC_API_URL=http://localhost:3001/api
DATABASE_URL=postgresql://localhost:5432/mydb_dev
```

```bash .env.development
# Development environment defaults
NEXT_PUBLIC_API_URL=http://localhost:3001/api
NEXT_PUBLIC_ENABLE_BETA_FEATURES=true
```

```bash .env.production
# Production environment
NEXT_PUBLIC_API_URL=https://api.example.com
NEXT_PUBLIC_ENABLE_ANALYTICS=true
NEXT_PUBLIC_ENABLE_BETA_FEATURES=false
```
</CodeGroup>

<Note>
  The `.env.local` file takes precedence over other environment files and should never be committed to version control.
</Note>

## Validation

Validate required environment variables at startup:

<CodeGroup>
```typescript validation.ts
const requiredEnvVars = [
  'NEXT_PUBLIC_API_URL',
  'DATABASE_URL',
  'JWT_SECRET'
];

requiredEnvVars.forEach((envVar) => {
  if (!process.env[envVar]) {
    throw new Error(`Missing required environment variable: ${envVar}`);
  }
});

export function validateEnv() {
  console.log('✓ All required environment variables are set');
}
```
</CodeGroup>
